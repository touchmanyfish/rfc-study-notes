# 15.2. Informational 1xx

[rfc 9110 15.2. Informational 1xx](https://www.rfc-editor.org/rfc/rfc9110.html#name-informational-1xx)

## rfc原文节选

###  client可能会收到一个或者多个1xx

```text
A client MUST be able to parse one or more 1xx responses received prior to a final response, even if the client does not 
expect one. A user agent MAY ignore unexpected 1xx responses.
```

### 1xx response 没有content和trailers

```kotlin
A 1xx response is terminated by the end of the header section; it cannot contain content or trailers.
```

## 当使用1xx时的交互流程

1. client第1次发送request
2. 服务器返回response(1xx)
3. client解析(**MUST**)response(1xx)，并处理(**MAY**)


4. client第2次发送request
5. 服务器返回response(1xx)
6. client解析(**MUST**)response(1xx)，并处理(**MAY**)


7. 服务器返回response(final)
8. client解析并处理response(final)

**需要注意的是:**

* client可能需要解析(**MUST**)1次或者多次reponse(1xx)
* 就算client没有期望收到1xx，它也要做好(**MUST**)解析1xx的准备
* client解析response(1xx)以后可以跳过处理
