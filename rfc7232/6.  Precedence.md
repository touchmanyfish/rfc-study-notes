# 6.  Precedence

[rfc 7232 6.  Precedence](https://www.rfc-editor.org/rfc/rfc7232#section-6)

这一小节定义request preconditions 的处理(MUST)顺序

## 优先级逻辑
```text
since "lost update" preconditions have more strict requirements than cache validation, a validated cache is more
 efficient than a partial response, and entity tags are presumed to be more accurate than date validators.
```
* "lost update" preconditions > cache validation > partial
  response
* entity tags > date validators

## 优先级处理
根据优先级逻辑，有如下优先级
1. If-Match
2. If-Unmodified-Since

3. If-None-Match
4. If-Modified-Since 

5. If-Range

**处理方式**:
* 从1.开始依次往后处理，成功则按照进入分支定义的跳转继续处理下一个，失败则按照precondition的定义进行返回
