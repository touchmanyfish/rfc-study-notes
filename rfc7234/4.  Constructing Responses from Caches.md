# 4.Constructing Responses from Caches
[rfc 7234 4.Constructing Responses from Caches](https://www.rfc-editor.org/rfc/rfc7234#section-4)

## 是否可以复用的条件
* A:URI    
* B:Method
* C:Vary
* D:Freshness/Cache Control Directives综合起来判断
* E:条件验证Validation(C/D/F导致的)
* F:Extensions

Extensions不覆盖任何条件
* 可以复用 = F and A and B and C and D and E 

Extensions覆盖了部分条件，假设全部覆盖
* 可以复用 = F   

1. 为了让条件为真，C/D/F判断时可能会出现需要进行Validation的情况
2. Extensions可以(**can**)覆盖前面所有条件
```
Note that any of the requirements listed above can be overridden by a cache-control extension;
```
3. 在D中Cache Control Directives可能会基于Freshness进行一些判断

## 复用时的相关处理

### 1.在没有进行validation的情况下复用，需要生成Age Header
```text
   When a stored response is used to satisfy a request without
   validation, a cache MUST generate an Age header field (Section 5.1),
   replacing any present in the response with a value equal to the
   stored response's current_age; see Section 4.2.3.
```

### 2.Cache必须(MUST)将unsafe method的request转发到origin server，并且还要执行Invalidation([section 4.4](https://www.rfc-editor.org/rfc/rfc7234#section-4.4))操作。
```text
   A cache MUST write through requests with methods that are unsafe
   (Section 4.2.1 of [RFC7231]) to the origin server; i.e., a cache is
   not allowed to generate a reply to such a request before having
   forwarded the request and having received a corresponding response.
   
     Also, note that unsafe requests might invalidate already-stored
   responses;
```

### 3.
```text
  When more than one suitable response is stored, a cache MUST use the
   most recent response (as determined by the Date header field).  It
   can also forward the request with "Cache-Control: max-age=0" or
   "Cache-Control: no-cache" to disambiguate which response to use.
```

### 1.满足rfc 7234 4.Constructing Responses from Caches中的基本要求
rfc中这部分主要将stored response被复用的基本条件，以及一些其他的处理

#### 如何满足
注意到本小节各条件之间使用“and”连接，表示：
```text
基本要求满足 = 条件1(true) and 条件2(true) and 条件3(true) ..
```
条件"the stored response is either"中使用了or，表明如果要为真三个至少满足1个。

#### 理解
```text
   o  The presented effective request URI (Section 5.5 of [RFC7230]) and
      that of the stored response match, and
```
```text
   o  the request method associated with the stored response allows it
      to be used for the presented request, and
```
某个method是否"allows",可以在其对应的(rfc 7231)详细定义中找到：
[GET](https://www.rfc-editor.org/rfc/rfc7231.html#page-24)
```text
The response to a GET request is cacheable; a cache MAY use it to
   satisfy subsequent GET and HEAD requests unless otherwise indicated
   by the Cache-Control header field (Section 5.2 of [RFC7234]).
```
1. 允许的情况:GET -> GET/GET -> HEAD
2. 这里使用“MAY”，表示cache就算认识GET请求，但是可以选择不支持GET对应response对缓存功能
3. unless otherwise”部分指基本条件还会被Cache Control(以及Extension)进行约束

[HEADE](https://www.rfc-editor.org/rfc/rfc7231.html#page-25)
```text
The response to a HEAD request is cacheable; a cache MAY use it to
   satisfy subsequent HEAD requests unless otherwise indicated by the
   Cache-Control header field (Section 5.2 of [RFC7234]).  A HEAD
   response might also have an effect on previously cached responses to
   GET; see Section 4.3.5 of [RFC7234].
```
1. 允许的情况:HEAD -> HEAD
2. "MAY"同理GET的2.
3. "indicated by"部分同理GET的3.

[POST](https://www.rfc-editor.org/rfc/rfc7231.html#page-25)
```text
   Responses to POST requests are only cacheable when they include
   explicit freshness information (see Section 4.2.1 of [RFC7234]).
   However, POST caching is not widely implemented.  For cases where an
   origin server wishes the client to be able to cache the result of a
   POST in a way that can be reused by a later GET, the origin server
   MAY send a 200 (OK) response containing the result and a
   Content-Location header field that has the same value as the POST's
   effective request URI (Section 3.1.4.2).
```

#### 举例
```text
// stored response
// cache have a clock available

GET /static/icon_logo.png
Date: Sat, 28 Mar 2026 10:00:00 GMT
Expires: Sun, 29 Mar 9026 10:00:00 GMT (指定一个巨大的过期时间)

// request 
// 时间： 2026年3月28日 14:00:00 (即 4 小时后)
GET /static/icon_logo.png

显示 fresh  = age < freshness_lifetime为真
```
上述例子满足[rfc 7234 4.Constructing Responses from Caches](https://www.rfc-editor.org/rfc/rfc7234#section-4)中的基本条件，所以
stored response可以被复用
### 2.Cache Control 指令对上述结果进行约束
```text
// stored response
Expires: Sun, 29 Mar 9026 10:00:00 GMT (指定一个巨大的过期时间)
ETag: "posts-v123"

// request
Cache-Control: max-age=0

//请求到达时stored response显然为fresh
```
如果只根据[rfc 7234 4.Constructing Responses from Caches](https://www.rfc-editor.org/rfc/rfc7234#section-4)中的基本要求
判断，stored response是符合复用要求的。但是请求使用了Cache-Control: max-age=0表示cache需要（假设有多个reponse，该验证哪一个？）



### 3.Cache Control Extension对上述结果进行约束

```
Note that any of the requirements listed above can be overridden by a
   cache-control extension; see Section 5.2.3.
```
注意，这里使用的是"can",表明当cache遇到extension与requirements冲突时，可以选择是否进行overridden，不是强制的。
