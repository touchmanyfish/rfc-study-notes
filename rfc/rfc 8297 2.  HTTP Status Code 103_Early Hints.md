# rfc 8297 2.  HTTP Status Code 103: Early Hints
[rfc 8297 2.  HTTP Status Code 103: Early Hints](https://www.rfc-editor.org/rfc/rfc8297.html#section-2)

[Hypertext Transfer Protocol (HTTP) Status Code Registry](https://www.iana.org/assignments/http-status-codes/http-status-codes.xhtml)

## 使用场景：利用服务器思考时间

103 的设计初衷是为了解决 Server Think-Time（服务器思考时间）的浪费问题：
* **传统流程**： 浏览器发请求 -> 服务器查数据库/渲染页面（耗时 100-500ms）-> 返回 200 OK -> 浏览器解析 HTML 后才开始加载 CSS/JS。

* **103 流程**： 浏览器发请求 -> 服务器立即回 103（带上关键资源的 Link header）-> 浏览器在服务器查数据时就开始预下载 CSS/JS -> 服务器查完数据回 200。


## 使用103时，client/server交互流程

参考[rfc 9110 15.2. Informational 1xx](https://www.rfc-editor.org/rfc/rfc9110.html#name-informational-1xx)

**例子:**
```text
   Client request:

     GET / HTTP/1.1
     Host: example.com


   Server response:

     HTTP/1.1 103 Early Hints
     Link: </style.css>; rel=preload; as=style
     Link: </script.js>; rel=preload; as=script

     HTTP/1.1 200 OK
     Date: Fri, 26 May 2017 10:02:11 GMT
     Content-Length: 1234
     Content-Type: text/html; charset=utf-8
     Link: </style.css>; rel=preload; as=style
     Link: </script.js>; rel=preload; as=script

     <!doctype html>
     [... rest of the response body is omitted from the example ...]
```
